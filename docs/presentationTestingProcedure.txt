시연 순서

- 미리 준비할 것 
	- 각 주소 복사해놓기
	- 바뀔 컨트랙트 V2 클레이튼 ide에서 준비

시크릿모드에서 카이카스 없을 때 시연

- 홈 화면 소개
	- 스크롤 액션
	- 반응형
	
- userRank1 으로 먼저 지갑 연결
	- 민트페이지와 유저정보 펼쳐서 whitelist 아닌것 확인

- userRank3 으로 변경해서 화이트리스트 표시되는것 확인

- ADMIN에서 화이트 리스트 조작
	- 추가 및 여러개 한번에 제거

- 스왑페이지
	- 토큰 추가
	- validation 보여주고
	- 클레이 => BTK 스왑

- 민팅페이지 이동
	- 가스비 제한으로 5개 제한
	- 화이트리스트 1개 진행
	- 일반 민팅 5개 진행

- 스테이킹 페이지
	- 가스비 제한으로 스테이킹 4개 제한
	- 스테이킹 및 언스테이킹
	- 리워드 새로고침 및 클레임ㅇ

- 콜렉션 페이지
	- sort로 민팅 여부 구분
	- filter로 메타데이터 구분

- 상세 페이지
	- 메타데이터 확인 가능
	- 앞 뒤로 이동
	- 슬라이드 기능

- 투표 페이지
	- rank3, deployer 글 쓰기 기능

	- deployer가 승인하고 투표 시작
		- 한번에 한 개 투표만 가능
		- 이미 투표한 사람은 투표 불가
		- 보팅 파워는 nft 총 보유량(스테이킹 포함)
		- 다른 승인 대기중은 승인 불가
		예상 Q) 컨트랙트에 안건 저장? -> 컨트랙트 상에는 안건 내용 등 저장되지 않고 ID만 저장

	- 투표 종료
		- 투표 종료로 상태 바뀜
		- 다른 안건 승인 가능

- 프록시 컨트랙트
	- 투표와 연계해서 투표에서 바꾸는 것으로 결정났다고 가정
	- V2로 upgrade 하고 data는 그대로인 것과 함수 변경된 것 시연

	userRank 바꾸는게 무난해 보인다
