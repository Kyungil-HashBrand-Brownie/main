## commit message rule
### 제목 (type: title)
-feat : 새로운 기능 추가
-fix : 버그 수정
-docs : 문서 관련
-refactor : 코드 리팩토링
-test : 테스트 코드
-chore : 빌드 업무 수정, 패키지 매니저 수정 등
-add : 기능도 아니고 문서도 아닌 나머지 기타적인 부분 추가
-delete : 삭제
-update : 버전 업데이트
-rename : 이름 변경
-move : 코드나 파일 이동
-modify : 수정
-correct : 문법 오류, 타입 변경, 오타 등
### 본문 
부연 설명
무엇을, 왜 변경했는가
### 꼬리말
이슈를 자동으로 종료하고, 필요한 경우 관련 이슈, 참고 이슈에 대한 정보 작성
-close : 일반 개발 이슈
-fix : 버그 픽스나 핫픽스 이슈
-resolve : 문의나 요청사항에 대응한 이슈
